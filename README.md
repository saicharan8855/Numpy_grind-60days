NumPy 60 Days Grind 🔥
Implementing NumPy functions from scratch using pure Python — no NumPy, no shortcuts.
Each day covers one function: a clean implementation, test cases, and edge/error cases.

📌 Project Overview
This project is a 60-day deep-dive into rebuilding NumPy's core functionality using only Python built-ins. The goal is to understand what NumPy does under the hood — how arrays are shaped, how operations broadcast, how reductions work — all without importing NumPy itself.

Each day follows a consistent structure:

Function: What it does and its mathematical definition

Implementation: Pure Python code (no NumPy)

Test Cases: Normal usage with expected outputs

Edge/Error Cases: Empty arrays, wrong types, mismatched shapes, boundary values


📈 What I Learned
How shapes and strides define array behavior

Why broadcasting works the way it does (rule-based axis alignment)

The math behind SVD, determinants, and matrix inverses

How sorting algorithms behave on already-sorted vs random data

Why std(ddof=1) differs from std(ddof=0) — Bessel's correction

Real appreciation for how much NumPy optimizes under the hood with C bindings


🧠 Skills Demonstrated
Pure Python logic — recursion, nested loops, list comprehensions

Mathematical rigor — implementing formulas from first principles

Error handling — type checks, shape validation, boundary conditions

Test-driven mindset — every function ships with passing and failing tests

Consistency — 60 commits, 60 days, zero skips
